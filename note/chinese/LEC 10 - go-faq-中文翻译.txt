问：defer关键字在其他语言中有吗？

答：Defer在Go中是新的。我们最初添加它是为了提供一种从恐慌中恢复的方法（参见规范中的"recover"），但结果证明它对于像"defer mu.Unlock()"这样的习语也很有用。后来，Swift也添加了defer语句。它显然受到Go的启发，但我不确定细节有多接近。

问：为什么不采用C++和Java中的类和OOP？

答：我们相信Go的面向对象编程方法，它更接近Smalltalk而不是Java/C++/Simula，更轻量级，使得更容易适应大型程序。我在2010年的Google I/O上谈到了这个。参见https://github.com/golang/go/wiki/GoTalks#go-programming获取视频和幻灯片的链接。

问：为什么多行结构定义需要尾随逗号？

答：最初它不需要，但所有语句都用分号终止。在Go公开发布后不久我们使分号可选。当我们这样做时，我们试图避免Javascript犯的使分号规则非常复杂和容易出错的错误。相反，我们有一个简单的规则：每行以隐式分号结束，除非最终标记是不可能结束语句的东西（例如加号或逗号）。这的一个效果是，如果你不在行上放置尾随逗号，它会得到一个隐式分号，这不能很好地解析。这是不幸的，在分号规则之前它不是那样的，但我们对不必一直输入分号感到如此高兴，以至于我们会忍受它。分号插入的原始提案在https://groups.google.com/d/msg/golang-nuts/XuMrWI0Q8uk/kXcBb4W3rH8J。也参见下一个答案。

问：为什么列表定义不一致，有些需要逗号有些不需要？

答：不需要逗号的那些需要分号，但那些分号正在自动插入（见前一个答案）。规则是语句用分号分隔，较小的语法片段用逗号分隔：

import "x";
import "y";

var x = []int{
	1,
	2,
	3,
}

当你提取一组导入时，你仍然有分号：

import (
	"x";
	"y";
)

var x = []int{
	1,
	2,
	3,
}

但后来当我们使分号可选时，分号从语句块中消失，留下逗号：

import (
	"x"
	"y"
)

var x = []int{
	1,
	2,
	3,
}

现在区别在于无和有之间，而不是两个不同的字符，这更明显。如果我们从一开始就知道分号是可选的，我想我们可能在更多的语法形式中使用它们，或者也许使某些形式接受逗号或分号。但在这一点上这似乎不太可能。

问：为什么Go将其while循环命名为"for"？

答：C既有while(cond) {}也有for(;cond;) {}。Go似乎不需要两个关键字来做同一件事。

问：最近似乎有很多新语言出现，包括Rust、D、Swift和Nim等。你从这些其他语言及其社区中学到了什么教训，希望你能将其纳入Go？

答：我确实关注那些语言的发展。我认为它们从Go中学到了东西，我希望我们也从它们中学到了东西。有一天我希望Go编译器能更好地推断所有权规则，或者在类型系统中甚至有轻量级所有权表达式。Javari、Midori、Pony和Rust是这里的灵感。我在https://research.swtch.com/go2017写了更多关于这方面的内容。

问：为什么专注于并发和goroutines？

答：我们从过去的经验知道，使用通道和轻量级进程的良好并发支持会使我们在Google构建的那种系统的编写更容易，正如我希望讲座所显示的那样。在https://golang.org/doc/faq#csp有更多解释，以及关于我们早期经验的一些背景在https://swtch.com/~rsc/thread/。
